元类实现curd中的删改查操作

与增加数据相同，只需要将用户传入的参数传入 sql 语句即可。

这三个操作要注意的两部分分别为：

  • 与插入操作不同，删除、查找与修改操作若未接受到字段名，不能以None替换，而应该略过；
  • 查找与修改操作中不仅仅需要条件参数，还需要“修改后的值”与“要返回的字段”，因此需要使用不同的传参形式，我这里查找使用元组传入要返回的字段，修改使用字典传入要修改的字段与值。

问题在于，四个操作分别对应四个类，需要将四个类统合为一个Model类以便于模型类继承不然就变成每个操作分别继承与不同的类这种麻烦的结构了。

最初的想法是将这四个类定义在继承于Model_metaclass元类的Model类中，并继承Model类，但发现这样的定义存在两个问题：

  • 四个类无法在Model类中被定义的同时继承Model类；
  • Model类与四个类的__init__()函数会出现各种各样的错误；
  • 可能会与Model_metaclass冲突。

目前不知道为什么会出现这些问题，以后会仔细研究一下。

我的解决方案是：

为每个操作分别定义一个函数（称为操作函数），函数内包含对应操作的类（此时类继承与基类不会失败）。模型类实例在调用时将参数传入操作函数，操作函数实例化一个操作类对象，并将参数传入类中，最后返回类实例即可。
