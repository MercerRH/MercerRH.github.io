使用Scrapy爬取微博内容并进行情感分析
发表于2020-12-08|更新于2021-01-20|爬虫
|字数总计:3.1k|阅读时长:13分钟|阅读量:

爬虫准备

采用scrapy进行爬虫，项目名为weibo

爬取目标分析

微博共有三个域名：微博简化版https://weibo.cn/、微博移动端https://m.weibo.cn/、微博PC端https://weibo.com/。三者界面的复杂程度依次提高，爬取难度依次提高。

我们采用简化版与移动端进行爬取。

确定爬取对象

爬取对象为热点话题的微博内容，具体要求如下：

  • 微博内容应该属于能引起不同道德情绪的热点话题（例如能引起愤怒的“清华学姐”）
  • 每条微博的发布者 id、性别、年龄、是否大V、是否大学生、发布时间
  • 用 MySQL 储存

移动端

各个目标的url

  • 话题：https://m.weibo.cn/search?containerid=100103type%3D1%26q%3D%23<经过编码的话题名称>%23

    扩展：url 编码有两种方式

    • encodeURI 编码：不会对特殊符号编码
    • encodeURIComponent 编码：会对特殊符号编码

    进行编码转换可以使用urllib包中parse模块中的quote()函数与unquote()函数分别进行编码与解码

  • 各个话题的实时微博内容：https://m.weibo.cn/api/container/getIndex?containerid=100103type%3D61%26q%3D%23<要搜索的话题>%23%26t%3D0&page_type=searchall&page=<页数>

  • 微博发布人的个人页面：https://m.weibo.cn/api/container/getIndex?containerid=230283<用户id>_-_INFO

  • 微博发布者个人界面json信息url：https://m.weibo.cn/api/container/getIndex?uid=<用户id>&luicode=10000011&lfid=100505<用户id>&type=uid&value=<用户id>&containerid=230283<用户id>

    删去不必要的查询字符串可得：https://m.weibo.cn/api/container/getIndex?uid=<用户id>&containerid=230283<用户id>

简化版

各个目标的url

  • 话题：https://weibo.cn/search/对该 url 进行POST请求，在data中包含下列字段：keyword=%23<要搜索的话题>%23&smblog=%E6%90%9C%E5%BE%AE%E5%8D%9A

    后来在翻页时发现，可以通过 GET 请求直接获取结果：https://weibo.cn/search/mblog?hideSearchFrame=&keyword=%23<关键词>%23&page=<页数>

  • 各个话题的实时微博内容：https://weibo.cn/comment/<微博id>，其中的微博 id 需从该微博的div标签中的id属性获取

  • 微博发布者的个人界面：https://weibo.cn/<发布者id>/info，发布者 id 同样可以从 html 中找到

分析爬取目标

移动端微博

对话题 url 进行请求时，返回的是json格式的数据，可以通过json模块进行解析。从数据格式可以看出，博文数据保存在data.cards中，每个数字字段中包含了一条微博的所有信息。

对json数据中每个数字字段进行分析后可以得到下列信息：

  • 发布时间：mblog.created_at
  • 微博id：mblog.id
  • 原始微博（未被转码）：mblog.text，可以通过判断结尾是否为</a>来判断该微博是否隐藏了全文
  • 微博文本（被转码后的微博）：mblog.raw_text，若未被隐藏全文则可直接从这里提取文本
  • 发布人所使用的设备：mblog.source
  • 该微博在简易版微博界面的 id：mblog.bid，由于在手机版微博全文界面提取全文过于复杂，因此可以跳转到简易版微博进行提取
  • 发布者 id：mblog.user.id
  • 发布者昵称：mblog.user.screen_name

简易版微博

对移动端微博的话题 url 进行请求首先得到的是一个框架界面，接着进行的请求中返回的json数据才是微博内容。而简易版微博则不然，所有内容都在 html 文件中，因此相较于前者更容易爬取，直接使用xpath提取即可。

从https://weibo.cn/search/搜索结果中可以提取到下列信息（xpath路径）：

  • 微博文本：//span[@class="ctt"]/text()
  • 发布者昵称：//div[@class="c" and @id]/div[1]/a[@class="nk"]/text()
  • 发布者个人页面 url：//div[@class="c" and @id]/div[1]/a[@class="nk"]/@href
  • 所有微博（选取微博内容与用户url在一起的div，即存在 id 属性的 div）：//div[@class="c" and @id]
  • 微博发布日期时间与设备：//div[@class="c" and @id]//span[@class="ct"]/text()

从用户个人界面https://weibo.cn/<用户id>/info可提取下列信息：

  • 用户个人信息：//div[text()="基本信息"]/following-sibling::div[1]/text()
  • 用户学习经历：//div[text()="学习经历"]/following-sibling::div[1]/text()
  • 用户其他信息：//div[text()="其他信息"]/following-sibling::div[1]/text()

数据库设计

对要爬取的数据进行分析：

  • 微博话题：

    话题 id（主键），话题名称

  • 微博：

    微博 id（主键），发布者 id（外键），所属话题 id（外键），微博文本，发布日期，发布者使用设备

  • 微博用户：

    用户 id（主键），用户昵称，用户性别，用户生日，是否大学生，所在地区，所在学校
